# 强前置条件约束

本文件只记录“代码里真实存在”的强约束。违反这些约束时，程序通常会直接 `exit(...)`。

## 1. 输入与维度约束

- 邻接矩阵必须是方阵
- 点数据必须是可解析的数值矩阵
- `--fileType` 仅支持：
  - `0` 点数据
  - `1` 邻接数据

## 2. 参数取值约束

- `--numpart >= 2`
- `--numInitOrderings > 0`
- `--knnAdjMode` 必须在 `0..3`
- `--gausssimAdjMode` 必须在 `0..1`
- `--policyPRCMetric` 必须在 `0..3`
- `--knnAdjK >= -1`
- `--tagLoc` 使用 `0/1/-1`

## 3. 初始顺序文件约束

当使用 `--initOrderFile` 时：

- 长度必须等于样本数
- 必须是 `0..N-1` 的一个排列
- 不能越界、不能重复、不能缺失

当使用 `--initLabelsFile` 时：

- 长度必须等于样本数
- 程序会按标签分组生成初始顺序

## 4. 功能开关约束（当前版本）

- `--useMultiLevelPRC 1`：当前不可用，程序会提示后退出
- `--useTransduction`：尚未实现（值会被忽略/重置）

## 5. 参数名兼容性注意

当前 Python 实现兼容两种写法：

- `--commentDelimiter`（推荐）
- `--commentDelimeter`（历史兼容）

## 6. Python 运行约束（当前主线）

- 主线入口是纯 Python：
  - `pinchRatioClustering.py`
  - `genSimMatrix.py`
- 推荐 Python 3.10+，`numpy` 为硬依赖
- `scipy` 不是硬依赖（稀疏输入会自动兼容）

## 7. 失败优先级排查

出现“运行即退出”时优先检查：

1. 输入文件存在性与格式
2. `fileType` 和数据格式是否匹配
3. 参数是否超出合法区间
4. 是否误开多层 PRC
5. Python 环境是否满足（版本与 `numpy`）
