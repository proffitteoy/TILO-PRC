# 实验设计原则

当前仓库的实验目标应该围绕两个问题：

1. Python 版本是否能稳定复现实验流程
2. 不同构图和策略参数会怎样影响聚类结果

## 推荐实验闭环

1. 固定输入数据
2. 固定随机种子
3. 固定构图参数
4. 固定 PRC/TILO 策略
5. 保存标签和顺序输出
6. 对比不同配置下的结果变化

## 优先控制的变量

### A. 输入类型

- 点数据：`--fileType 0`
- 邻接图：`--fileType 1`

### B. 构图方式

- `--pointSimilarity 1`：Gaussian
- `--pointSimilarity 2`：KNN

### C. 切分指标

- `--policyPRCMetric 0`：PinchRatio
- `--policyPRCMetric 3`：NCut

建议先比较这两个主流指标，再看实验性指标。

### D. 初始化方式

- 固定 `--initOrderFile`
- 或固定 `--seed`
- 或使用 `--numInitOrderings` 做多次初始化

## 推荐基线

### 基线 1：固定顺序验证主流程

```powershell
python pinchRatioClustering.py --dataInput datasets/graphs/d1.txt --fileType 1 --adjNodeOffset 0 --initOrderFile datasets/graphs/d1_initialorder.txt --numpart 2 --saveOrder 0 --saveLabels 0
```

### 基线 2：点数据 + 标签可评估

```powershell
python pinchRatioClustering.py --dataInput datasets/iris/iris_all.txt --fileType 0 --tagLoc 1 --pointSimilarity 2 --knnAdjMode 3 --knnAdjK -1 --numpart 3 --seed 42
```

### 基线 3：多次初始化择优

```powershell
python pinchRatioClustering.py --dataInput datasets/iris/iris_all.txt --fileType 0 --tagLoc 1 --pointSimilarity 2 --knnAdjMode 3 --numpart 3 --seed 42 --numInitOrderings 10 --saveMultipleRuns 1
```

## 结果记录建议

每次实验至少记录：

- 输入文件
- 完整命令
- Python 版本
- 关键参数
- 输出文件路径
- 标签摘要
- 异常信息

## 对结果的合理预期

当前更适合验证的是：

- 流程是否可运行
- 输出是否稳定
- 参数变化是否带来合理差异

当前不应直接把少量样例跑通理解成“已经严格证明与历史 C++ 结果完全一致”。
